# Премиальные ярлыки ПТО: тест и публикация

## Что теперь есть

В корне проекта созданы 5 файлов:

1. `create-desktop-shortcuts.bat` — автоматическое создание брендированных ярлыков на рабочем столе.
2. `set-online-mode.bat` — включает рабочий онлайн-режим (`VITE_FORCE_DEMO=0`).
3. `test-local-site.bat` — локальный production-тест (без публикации в интернет).
4. `deploy-site.bat` — безопасная публикация (с обязательными проверками).
5. `set-demo-mode.bat` — сервисный demo-режим (`VITE_FORCE_DEMO=1`).

## Названия ярлыков на рабочем столе

После запуска `create-desktop-shortcuts.bat` создаются:

1. `ПТО · 1. Включить Онлайн`
2. `ПТО · 2. Локальный Тест`
3. `ПТО · 3. Безопасная Публикация`
4. `ПТО · D. Включить Демо`

## Что делает локальный тест (`test-local-site.bat`)

1. Переходит в `web/frontend`.
2. Выполняет production-сборку (`npm run build`).
3. Запускает локальный просмотр сайта (`npm run preview -- --host`).
4. Автоматически открывает браузер на `http://localhost:4173`.

Лог записывается в `local-test-last.log`.

## Что делает безопасная публикация (`deploy-site.bat`)

1. Проверяет, что проект открыт как git-репозиторий.
2. Проверяет, что вы на publish-ветке (`main` или `master`).
   - если активна другая ветка, скрипт предложит авто-переключение;
   - если есть несохраненные изменения, публикация будет остановлена с понятной подсказкой.
3. Проверяет, что demo-режим выключен (`VITE_FORCE_DEMO=0`).
4. Собирает фронтенд (`web/frontend`, `npm run build`).
5. Собирает Firebase Functions (`web/firebase/functions`, `npm run build`).
6. Автокоммитит изменения (если они есть).
7. Просит явное подтверждение `YES` перед отправкой в интернет.
8. Сохраняет точку отката в `last-stable-commit.txt`.
9. Делает `git push`.

Лог записывается в `deploy-last.log`.

## Как обновить ярлыки

1. Откройте папку `C:\Users\Surovcev\PTO_Project`.
2. Дважды кликните `create-desktop-shortcuts.bat`.
3. На рабочем столе появятся/обновятся брендированные ярлыки ПТО.

## Ваш рабочий сценарий (без терминала)

1. Запускаете `ПТО · 1. Включить Онлайн`.
2. Запускаете `ПТО · 2. Локальный Тест`.
3. Проверяете сайт локально.
4. Если все хорошо — запускаете `ПТО · 3. Безопасная Публикация`.
5. Подтверждаете публикацию словом `YES`.
6. После завершения скрипты ждут нажатия клавиши и не закрываются сразу.

## Что вы увидите в окне

- Крупные этапы `Шаг 1/2/3`, чтобы было понятно, что делает система.
- Понятное сообщение об ошибке, если что-то не так.
- Пауза в конце, чтобы вы успели прочитать результат.
- Единый стиль названий и описаний ярлыков.

## Если что-то пошло не так

1. Откройте `deploy-last.log` или `local-test-last.log`.
2. Найдите последнюю строку `ERROR: ...`.
3. Исправьте причину и снова запустите нужный ярлык.
