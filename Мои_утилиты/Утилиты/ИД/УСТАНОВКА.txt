Утилита «ИД»

Установка зависимостей (из этой папки):

  pip install -r requirements.txt

Нужны PyMuPDF (fitz), openpyxl, python-docx.

Для распознавания сканов (OCR пустых страниц PDF):
  pip install pytesseract
и установите движок Tesseract OCR for Windows (https://github.com/UB-Mannheim/tesseract/wiki),
либо добавьте путь к tesseract.exe в PATH.

Данные проекта: папка «Данные», файл текущий_проект_рд.sqlite.

Текстовый черновик справа строится из базы РД; если символов текста нет — PDF похож на скан без слоя текста (нужен OCR или другой источник данных).

Комплект ИД «как в образце»

1. Отредактируйте файл «Данные/комплект_id.json»: шифр, блоки заказчика/объекта, основание КС,
   список позиций реестра (наименование, обозначение), путь «папка_образца» и флаг
   «копировать_все_файлы_из_образца».

2. В утилите нажмите «Сформировать комплект ИД…» и укажите пустую выходную папку.

   • Если копирование включено — сначала копируются все файлы из образца (АОСР, ведомости, сертификаты и т.д.),
     затем заново записываются титул и реестр по шаблонам из «Шаблоны/».

   • Если выключено — создаются только «0. Титульный лист.docx» и файл реестра с именем из JSON.

3. В «замены_в_титуле» задайте пары [было, стало] для замены текста в скопированном титуле (например старый шифр → новый).

В папке «Шаблоны» лежат образцы реестра и титула, по которым программа правит Excel и Word.
