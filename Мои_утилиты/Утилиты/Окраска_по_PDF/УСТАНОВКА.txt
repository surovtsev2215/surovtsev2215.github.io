Утилита «Окраска по PDF» (авто-режим) использует пакет PyMuPDF.



Установка (один раз), из папки этой утилиты в командной строке:



  pip install -r requirements.txt



Или из корня хаба:



  pip install -r "Мои_утилиты/Утилиты/Окраска_по_PDF/requirements.txt"



После установки перезапустите «Запустить.py» или нажмите F5 в хабе.



Отчёт Microsoft Excel (.xlsx)

------------------------------



В меню «Экспорт…» доступен пункт «Отчёт Excel (.xlsx)…»: оформленная книга с листами «Сводка», «Ведомость» и «Проверки» (таблицы, шапка с объектом из PDF/имени файла, при необходимости укажите объект вручную в поле под каталогом).



Если пакет не найден:



  pip install openpyxl



Как работает

------------

Выберите цель расчёта:



1) Металл по ведомости (CSV)

   • Рядом с утилитой: «профили_база.csv» (общая база: балки по примеру + сетка листов B×t мм) и
     «профили_м2_на_пм.csv» (ваш проектный слой: те же столбцы; при совпадении ключа значения из
     локального файла заменяют базу).

   • Формат строки: ключ;м2_на_пм [;кг_на_пм]. Строки, начинающиеся с #, комментарии.
     Третья колонка (кг на погонный метр) опциональна, но нужна для самопроверок по массе и «площади по весу».
     Для двутавров и нестандартных узлов возьмите кг/м из той же ведомости (масса / длина в метрах) или из сортамента.

   • Ключ для двутавров: строка сечения без пробелов (например двутавр40к2) или габарит 194x10.

   • Значения м²/п.м вручную сверяйте с вашим нормативом / сортаментом. Для листов в базе добавлена
     крупная сетка размеров: м²/м оценены интерполяцией от опорных строк, кг/м = 7850·B·t·10⁻⁶ (B,t в мм).

   • Утилита ищет блок «Спецификация деталей / Specification», разбирает строки (позиция, кол-во, длина мм, марка стали, профиль) и считает:

     площадь строки = (м²/п.м) × (длина в метрах) × кол-во.

   • Если для листа B×t в CSV нет строки, но размеры распознаны в ячейке профиля, подставляется запасная
     оценка м²/м (интерполяция по опорным точкам) и кг/м = 7850·B·t·10⁻⁶; в служебном поле строки будет пометка.

   • Строки без распознанного профиля и без ключа в CSV попадают с м²=0 — дополните каталог или проверьте текст PDF.
   • В таблице результатов позиции сгруппированы по марке сборки; марка наследуется от предыдущих строк ведомости, при отсутствии в тексте возможна подсказка из имени PDF (например …_Б1_2… → Б1-2).

   • Блок «Ведомость отправочных элементов / Statement assembly»: если в тексте PDF найдены марка и количество комплектов (Q-ty),
     площади и массы строк умножаются на это число (спецификация деталей задаёт один комплект).
     Листовые детали в колонке профиля показываются как B×L×t мм (ширина × длина из графы длины × толщина из ячейки B×t).

   • Меню «Ещё» — каталог из интернета: HTTPS-URL на CSV (тот же формат). Файл настроек «каталог_url.json»,
     кеш «профили_база_облако.csv», метаданные «каталог_облако_meta.json». Пока в настройках не включено
     «Использовать удалённый каталог», кеш в расчёт не подмешивается. Слой: база → облако → профили_м2_на_пм.csv.
     Пункт «Ключи без м²/м в каталоге (CSV)» после анализа — что дописать в справочник.

   • Можно указать другой CSV кнопкой «Файл каталога…».
   • После «Анализ» в режиме металла: блок «Самопроверки» (линтер CSV, согласованность масс из PDF с каталогом,
     сверка суммы м² с явной площадью в тексте чертежа при наличии). Индекс качества 0–100 — эвристика разбора, не сертификат.
     «Допуск %», «Строгий режим», экспорт «Отчёт проверок JSON», копирование сводки.



2) Площадь листа PDF

   • Режим как раньше: спецификация / размеры в тексте / штамп «Формат: A0–A4» (ISO 216) и поля по умолчанию; пары сечений проката не подменяют лист.



Превью чертежа не показывается — список файлов, прогресс и таблица.



Ограничения

------------

• Разбор ведомости v1 рассчитан на типичный порядок полей в тексте PDF; при другом шаблоне экспорта столбцов может понадобиться доработка правил.

• Сканы без текстового слоя: ведомость не извлечётся.

• Полный ГОСТ в утилиту не встроен — только ваш CSV.



Вкладка «Ассистент» (чат с LLM)

--------------------------------

Нужны интернет и ключ OpenAI-совместимого API.

1) Файл assistant_config.json рядом с утилитой (рядом с Запуск.py), пример:

   {"api_key":"sk-...","base_url":"https://api.openai.com/v1","model":"gpt-4o-mini"}

   Или переменная окружения OPENAI_API_KEY — ключ можно не дублировать в JSON.

   Не коммитьте ключ в git (assistant_config.json в .gitignore репозитория).

2) Вкладка «Ассистент»: нажмите «Выбрать PDF…» — это текущий состав проекта (один или несколько файлов),
   список синхронизируется с вкладкой «Расчёт АКЗ». Отдельной вкладки «Оси и зоны» нет: зоны настраиваются
   через чат ассистента и меню «Ещё» на вкладке «Расчёт АКЗ» (пресет осей, шаблоны КМ/КМД, общий шаблон на диск,
   экспорт свода CSV, добавление зоны).

   Рядом с выбранными PDF программа ищет JSON проекта: для одного файла — «<имя_pdf>_окраска_проект.json»
   рядом с PDF; для нескольких PDF в одной папке — «_окраска_несколько_pdf_проект.json» в этой папке.
   Если файл есть, из него подтягиваются цель расчёта, каталог (если файл на месте), название отчёта,
   зоны и сохранённые результаты только для тех PDF, которые вы выбрали (выбранный список PDF не заменяется списком из JSON).

   Полное открытие сессии по любому JSON по-прежнему в меню «Ещё» → «Открыть сессию (JSON)…».

3) Напишите, что сделать (оси 1–5, режим металла/листа, пересчёт, сохранить проект, экспорт Excel).

На сервер уходят только имена PDF и текущие зоны — содержимое PDF не отправляется.

Сохранение проекта JSON v4 включает поле axis_groups (зоны этого объекта).


Ключ профиля и каталог (гибрид без «выдуманных» м²)

--------------------------------

• Коэффициенты м²/м и кг/м берутся только из CSV; при расхождении с массой PDF программа может переопределить
  ключ профиля среди кандидатов из того же каталога (сверка по кг/м). Опционально — запрос к LLM только
  как «ничья» между похожими ключами (ответ строго один из переданных ключей).

• В assistant_config.json (опционально):

  • "catalog_key_mass_fit_resolve": true  — включена авто-сверка ключа по кг/м (по умолчанию true, если ключ не задан).
  • "catalog_key_disambiguate_llm": false  — LLM на ничьях (по умолчанию false; включайте осознанно, нужен api_key).

  Дополнительно: "mass_fit_margin_rel" (напр. 0.012), "tiebreak_llm_max_groups", "tiebreak_llm_timeout_sec".

• В JSON проекта (*_окраска_проект.json) можно задать "profile_aliases_raw_to_key":
  объект «канонический raw → ключ CSV» для постоянных соответствий. Рядом с утилитой допускается файл
  profile_aliases_pending.json с тем же полем для черновиков.

• Общий каталог компании можно подтягивать по HTTPS — см. каталог_url.json и README в каталог_удалённый.py.
  Локальный профили_м2_на_пм.csv по-прежнему перекрывает слои при слиянии.

• При сохранении проекта / экспорт JSON поле merged_diagnostics может содержать вложение
  profile_key_disambiguation: там перечислены строки, где ключ подставлен по кг/м или LLM-ничье,
  с краткими rank_scores и флагами отката.
